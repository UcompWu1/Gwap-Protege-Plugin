uComp Crowdsourcing Validation Plugin

A plugin for Protege which validates a part of an ontology (or a complete ontology) with the help of the uComp crowdsourcing API.

Version information

v1.1.1 - 02/07/2014

  • Bugfixing and improvements for communication with uComp-API.

v1.1.0 - 18/06/2014

  • Individual Validation: Added feature for validating all individuals which are of the same type (OWL class) as the currently selected individual. All validation tasks are sent in a single job.
  • Object Property Validation: Added feature for validating all object properties in a single job (batch validation).
  • Relation Validation: Refined feature for validating all relations. All tasks are now sent in a single job.

v1.0.3 - 28/05/2014

  • Bugfixing for all validations for feature for calculating expected job costs.
  • GUI optimization for all validations: status messages are now displayed in HTML.

v1.0.2 - 21/05/2014

  • Bugfixing for class validation for feature for calculating expected job costs.
  • Buxfixing for class validation when selecting root class Thing.
  • GUI optimization for class validation: status messages are now displayed in HTML.

v1.0.1 - 14/05/2014

  • Added detailed error messages in the User Interface for problems with missing/malformed uComp-API settings-files.
  • Added/refined tooltip texts for GUI elements.
  • Replaced the "Send to CrowdFlower/uComp-Quiz" checkbox with radio buttons.
  • Added new feature for calculating expected job costs.

v1.0.0 - 09/05/2014

  • Initial version. See link to documentation below for detailed information.

See uComp Crowdsourcing Validation Plugin Documentation for further information.
